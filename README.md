# Claude Usage Tracker

A VS Code extension that tracks your [Claude Code](https://claude.ai/code) CLI token usage and estimated cost, surfaced directly inside VS Code.

Built using the [Ralph Wiggum](https://github.com/alfaloo/ralph-wiggum) agentic coding framework.

---

## Features

- **Status bar item** — always-visible `$(graph) Claude: $0.45 · 125k tokens` showing today's totals
- **Dashboard panel** — click the status bar item (or run `Claude Usage: Open Dashboard`) to open a full analytics view
- **Real-time updates** — watches `~/.claude/projects/` for new session files; also polls every 60 seconds
- **No API calls** — reads only local JSONL files written by Claude Code; no network requests

### Dashboard tabs

| Tab | What it shows |
|-----|--------------|
| Today | Hourly bar chart for the current calendar day |
| This Week | Daily bars for the last 7 days |
| This Month | Daily bars for the last 30 days |
| All Time | All recorded sessions |

Each tab includes:
- Summary cards: Input Tokens, Output Tokens, Cache Tokens, Estimated Cost
- Bar chart (pure HTML/CSS, no external library)
- Model breakdown table (Sonnet / Opus / Haiku)
- Project breakdown table (by working directory)

---

## How it works

Claude Code writes a JSONL file for every session under `~/.claude/projects/<project-hash>/<session-id>.jsonl`. Each line is a JSON object; lines where `type === "assistant"` carry a `message.usage` field with token counts.

```
~/.claude/projects/
  -Users-ivan-Downloads-my-project/
    abc123.jsonl          ← session files
    def456.jsonl
```

The extension recursively scans all `.jsonl` files, parses usage entries, and calculates cost using Anthropic's published per-million-token rates.

### Token pricing (per million tokens)

| Model | Input | Output | Cache Read | Cache Write |
|-------|-------|--------|------------|-------------|
| claude-sonnet-* (default) | $3.00 | $15.00 | $0.30 | $3.75 |
| claude-opus-* | $15.00 | $75.00 | $1.50 | $18.75 |
| claude-haiku-* | $0.80 | $4.00 | $0.08 | $1.00 |

## Project structure

```
claude-usage-tracker/
├── src/
│   ├── extension.ts        # Entry point — activates extension, wires components
│   ├── types.ts            # Shared UsageEntry interface
│   ├── usageParser.ts      # Recursively reads ~/.claude/projects/**/*.jsonl
│   ├── costCalculator.ts   # Per-model pricing, formatCost(), formatTokens()
│   ├── statusBar.ts        # Status bar item (left-aligned, opens dashboard on click)
│   └── dashboardPanel.ts   # Singleton WebviewPanel manager
├── webview/
│   ├── App.tsx             # React dashboard UI (tabs, charts, tables)
│   └── index.html          # Webview HTML template
├── dist/                   # Built output (generated by esbuild, git-ignored)
│   ├── extension.js        # Bundled extension host code (CommonJS, Node)
│   └── webview.js          # Bundled React app (IIFE, browser)
├── esbuild.mjs             # Two-step build: extension + webview bundles
├── package.json
└── tsconfig.json
```

### Key files

**`src/usageParser.ts`** — walks the `~/.claude/projects/` directory tree, reads every `.jsonl` file, and returns a flat `UsageEntry[]` array. Malformed lines are silently skipped.

**`src/costCalculator.ts`** — `calculateCost(entry)` multiplies token counts by per-million rates based on model string matching (`haiku`, `opus`, or sonnet default). Also exports `formatCost()` and `formatTokens()` display helpers.

**`src/statusBar.ts`** — `UsageStatusBar` wraps a VS Code status bar item. `update(entries)` filters to today's entries and sets the display text.

**`src/dashboardPanel.ts`** — `DashboardPanel.createOrShow()` opens (or reveals) the webview panel. Sends `{ type: 'update', entries }` via `postMessage` whenever data refreshes.

**`webview/App.tsx`** — React app rendered inside the VS Code webview sandbox. Listens for `message` events from the extension host, filters entries by the active tab, and renders summary cards, a pure-CSS bar chart, and two breakdown tables.

**`src/extension.ts`** — ties everything together:
1. Creates the status bar and loads data on activation
2. Registers the `claudeUsage.openDashboard` command
3. Sets up a 60-second `setInterval` refresh
4. Uses `fs.watch({ recursive: true })` on `~/.claude/projects/` for live updates

## Getting started

### Prerequisites

- [VS Code](https://code.visualstudio.com/) 1.85+
- Node.js 18+ and npm
- [Claude Code CLI](https://claude.ai/code) (must have at least one session recorded)

### Install and run

```bash
git clone https://github.com/hiivan/claude-usage-tracker.git
cd claude-usage-tracker
npm install
npm run build
```

Then open the project in VS Code and press `F5` to launch the **Extension Development Host**. The status bar item appears immediately.

### Build commands

| Command | Description |
|---------|-------------|
| `npm run build` | One-shot production build |
| `npm run watch` | Incremental rebuild on file changes |
| `npm run package` | Package as `.vsix` for local install |

## How Ralph Wiggum built this

This extension was implemented using [Ralph Wiggum](https://github.com/alfaloo/ralph-wiggum), a CLI-driven agentic coding framework. The workflow:

1. `ralph init claude-usage-tracker` — created the project scaffold
2. `ralph enrich claude-usage-tracker` — an agent read the spec and generated a 10-task `tasks.json`
3. `ralph execute claude-usage-tracker --base master` — spawned a fresh Claude agent per task, each committing its own implementation

Ralph avoids context window degradation by starting each agent with only the spec, task list, and completion history — no accumulated conversation state.

---

## Data privacy

All data stays local. The extension reads only files already written by the Claude Code CLI to `~/.claude/projects/`. No tokens, costs, or session data are ever sent anywhere.
